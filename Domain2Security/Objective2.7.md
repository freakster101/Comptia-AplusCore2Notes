# 2.7 Given a scenario, apply workstation security options and hardening techniques

# Data-at-Rest Encryption

Protects stored data by encrypting files or drives to prevent unauthorized access.
Example: Encrypting a laptop SSD using BitLocker.

# Password Considerations

## Length

Longer passwords are harder to crack and improve security.
Example: Using a 14-character password instead of 6 characters.

## Character Types

Strong passwords use uppercase, lowercase, numbers, and symbols.
Example: `J@gdish2026!`

## Uniqueness

Each account should use a different password to reduce compromise risks.
Example: Different passwords for Gmail and banking accounts.

## Complexity

Complex passwords combine multiple character patterns to increase difficulty.
Example: Using random words, symbols, and numbers together.

## Expiration

Passwords should be changed periodically to reduce long-term exposure.
Example: Company policy requiring password changes every 90 days.

# BIOS/UEFI Passwords

Protects firmware settings and prevents unauthorized system boot changes.
Example: Setting a BIOS administrator password.

# End-User Best Practices

## Use Screensaver Locks

Automatically locks the workstation after inactivity.
Example: Screen locking after 10 minutes idle.

## Log Off When Not in Use

Prevents unauthorized access to active user sessions.
Example: Logging off before leaving the office.

## Secure/Protect Critical Hardware

Physically securing important devices to prevent theft or tampering.
Example: Using a laptop cable lock in public areas.

## Secure Personally Identifiable Information (PII) and Passwords

Protecting sensitive personal and authentication data from exposure.
Example: Avoiding writing passwords on sticky notes.

## Use Password Managers

Stores and manages passwords securely in encrypted vaults.
Example: Using Bitwarden or LastPass for account passwords.

# Account Management

## Restrict User Permissions

Giving users only the minimum access required for their tasks.
Example: Standard users without administrator rights.

## Restrict Log-in Times

Limiting when users can access systems.
Example: Employees allowed to log in only during work hours.

## Disable Guest Account

Turning off unnecessary guest access to improve security.
Example: Disabling Windows Guest account.

## Use Failed Attempts Lockout

Locks accounts after repeated incorrect password attempts.
Example: Account locked after five failed logins.

## Use Timeout/Screen Lock

Automatically locks systems after inactivity.
Example: Workstation locking after 15 minutes idle.

## Apply Account Expiration Dates

Automatically disables accounts after a specific date or project completion.
Example: Temporary employee account expiring after contract end.

# Change Default Administrator’s User Account/Password

Renaming or changing default administrator credentials to reduce attack risks.
Example: Renaming the default Administrator account in Windows.

# Disable AutoRun

Prevents automatic execution of files from removable media.
Example: USB drives not launching programs automatically.

# Disable Unused Services

Turning off unnecessary services to reduce vulnerabilities and resource usage.
Example: Disabling unused Remote Registry service.
