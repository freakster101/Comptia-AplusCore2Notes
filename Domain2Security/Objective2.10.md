# 2.10 Given a scenario, apply security settings on SOHO wireless and wired networks

# Router Settings

## Change Default Passwords

Replacing factory default router credentials to prevent unauthorized access.
Example: Changing admin/admin to a strong custom password.

## IP Filtering

Allows or blocks devices based on their IP addresses.
Example: Blocking a specific device from internet access.

## Firmware Updates

Installing router software updates to improve security and performance.
Example: Updating router firmware to patch vulnerabilities.

## Content Filtering

Restricts access to specific websites or online content.
Example: Blocking social media websites on a home network.

## Physical Placement/Secure Locations

Placing networking devices in safe and optimal locations to improve security and signal strength.
Example: Keeping the router inside a locked office instead of a public hallway.

## Universal Plug and Play (UPnP)

Automatically allows devices to configure network ports for communication.
Example: Gaming console automatically opening ports for online gaming.

## Screened Subnet

A separate network segment used to isolate public-facing systems from the internal network.
Example: Placing a web server in a DMZ network.

## Configure Secure Management Access

Securing router administration using encrypted protocols and strong credentials.
Example: Using HTTPS instead of HTTP for router management.

# Wireless Specific

## Changing the Service Set Identifier (SSID)

Modifying the wireless network name for identification and security purposes.
Example: Renaming Wi-Fi from “DefaultRouter” to “HomeNetwork.”

## Disabling SSID Broadcast

Hides the wireless network name from automatic discovery.
Example: Hidden Wi-Fi network not appearing in device scan results.

## Encryption Settings

Protects wireless traffic using encryption protocols.
Example: Configuring WPA3 encryption on a Wi-Fi router.

## Configuring Guest Access

Creates a separate network for visitors without exposing the main network.
Example: Guest Wi-Fi for customers in a café.

# Firewall Settings

## Disabling Unused Ports

Closing unnecessary network ports to reduce security risks.
Example: Disabling unused Telnet port access.

## Port Forwarding/Mapping

Directs incoming network traffic to a specific internal device or service.
Example: Forwarding port 3389 for Remote Desktop access.
