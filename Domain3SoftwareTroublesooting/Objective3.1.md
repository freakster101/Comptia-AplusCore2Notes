# 3.1 Given a scenario, troubleshoot common Windows OS issues

# Blue Screen of Death (BSOD)

A critical Windows error that forces the system to stop to prevent damage.
Example: A faulty driver causing a BSOD during startup.

# Degraded Performance

The system becomes slower than normal due to resource or hardware issues.
Example: High CPU usage causing slow application response.

# Boot Issues

Problems that prevent Windows from starting properly.
Example: Corrupted boot files causing startup failure.

# Frequent Shutdowns

The computer unexpectedly powers off or restarts.
Example: Overheating CPU causing automatic shutdowns.

# Services Not Starting

Required Windows services fail to launch during startup or operation.
Example: Print Spooler service failing to start.

# Applications Crashing

Programs unexpectedly close or stop responding.
Example: Microsoft Word crashing due to a damaged add-in.

# Low Memory Warnings

Windows detects insufficient available RAM for running applications.
Example: Multiple applications consuming nearly all system memory.

# USB Controller Resource Warnings

Windows reports insufficient resources for connected USB devices.
Example: Too many USB devices connected to a single controller.

# System Instability

The operating system behaves unpredictably with freezes or errors.
Example: Faulty RAM causing random crashes and lockups.

# No OS Found

The system cannot locate a bootable operating system.
Example: Failed hard drive preventing Windows from loading.

# Slow Profile Load

User accounts take an unusually long time to log in.
Example: Corrupted user profile causing delayed login.

# Time Drift

The system clock gradually becomes inaccurate over time.
Example: Weak CMOS battery causing incorrect system time.
