# 1.10 Given a scenario, install applications according to requirements

# System Requirements for Applications

## 32-bit vs. 64-bit Dependent Application Requirements
Applications must match the operating system architecture for compatibility and performance.  
Example: Installing a 64-bit video editor on 64-bit Windows.

## Dedicated vs. Integrated Graphics Card
Some applications require a dedicated GPU instead of integrated graphics for better performance.  
Example: Gaming software requiring an NVIDIA graphics card.

## Video Random-Access Memory (VRAM) Requirements
Graphics-intensive applications require enough VRAM for rendering and display tasks.  
Example: A 3D design application requiring 4GB VRAM.

## RAM Requirements
Applications need a minimum amount of memory to run properly.  
Example: Photoshop requiring at least 8GB RAM.

## Central Processing Unit (CPU) Requirements
Applications may require specific CPU speed, cores, or instruction support.  
Example: Video rendering software needing a quad-core processor.

## External Hardware Tokens
Some applications require physical authentication devices for access or licensing.  
Example: Banking software using a USB security token.

## Storage Requirements
Applications require sufficient disk space for installation and operation.  
Example: A game requiring 100GB of free SSD space.

## Application to OS Compatibility
Applications must support the installed operating system version.  
Example: Older accounting software not working on Windows 11.

# Distribution Methods

## Physical Media vs. Mountable ISO File
Applications can be installed from discs or virtual ISO image files.  
Example: Installing Windows from a bootable ISO file.

## Downloadable Package
Applications distributed through internet downloads.  
Example: Downloading Chrome installer from Google.

## Image Deployment
Preconfigured application images deployed to multiple systems simultaneously.  
Example: IT deploying standard software to office computers.

# Impact Considerations for New Applications

## Device
New applications may affect hardware performance or compatibility.  
Example: CAD software slowing down older PCs.

## Network
Applications may increase bandwidth usage or require network access.  
Example: Cloud backup software consuming internet bandwidth.

## Operation
Applications can affect system stability or workflow processes.  
Example: Antivirus software causing slower startup times.

## Business
Software installations may impact productivity, licensing, or company costs.  
Example: Purchasing enterprise licenses for office software.
